# План презентации

## 1. Тема и бизнес-задача

- Тема: автоматизированный ML-пайплайн для прогнозирования задержек общественного транспорта.
- Бизнес-задача: заранее оценивать риск задержки.
- Польза:
  - пассажиры могут точнее планировать поездку;
  - транспортный оператор может видеть проблемные маршруты;
  - сервис может улучшать качество расписания и информирования.

## 2. Данные и признаки

- Датасет: `data/public_transport_delays.csv`.
- Целевая переменная: `delayed`.
- Тип задачи: бинарная классификация.
- Используемые признаки:
  - маршрут;
  - станция;
  - погодные условия;
  - трафик;
  - события;
  - расписание;
  - дата и время.

## 3. Архитектура ML-пайплайна

- Загрузка данных из CSV.
- Удаление leakage-признаков — признаков, которые могут содержать утечку информации о целевой переменной.
- Добавление date/time-признаков — признаков на основе даты и времени.
- Разделение данных на train/test.
- Preprocessing через `Pipeline` и `ColumnTransformer`.
- Сохранение модели, метрик и отчётов.

## 4. Автоматизация обучения и подбор модели

- `SelectKBest` — автоматический отбор наиболее полезных признаков.
- `GridSearchCV` — подбор гиперпараметров.
- Сравнение моделей:
  - `LogisticRegression`;
  - `RandomForestClassifier`;
  - `GradientBoostingClassifier`.
- Лучшая модель выбирается автоматически по метрикам качества.

## 5. Метрики и сравнение моделей

- Используемые метрики:
  - `accuracy`;
  - `precision`;
  - `recall`;
  - `f1`;
  - `roc_auc`.
- Confusion matrix — матрица ошибок.
- Графики и таблица сравнения моделей сохраняются в `reports/`.

## 6. Docker, CI/CD и мониторинг

- Docker обеспечивает воспроизводимый запуск проекта.
- Bind mounts — подключение локальных папок к контейнеру:
  - `data`;
  - `reports`;
  - `models`;
  - `mlruns`.
- GitHub Actions запускает тесты и проверяет сборку Docker image.
- Мониторинг:
  - качество модели через метрики;
  - CPU/RAM через `psutil`;
  - внешняя проверка через `docker stats` и `docker compose logs`.

## 7. Выводы для бизнеса и проекта

- Проект показывает воспроизводимый процесс обучения ML-модели.
- MLflow помогает сравнивать эксперименты и сохранять историю запусков.
- Docker упрощает повторный запуск пайплайна в одинаковом окружении.
- Отчёты, графики и сохранённая модель упрощают аудит результата.
- Такой подход можно расширить для регулярного переобучения модели на новых данных.
